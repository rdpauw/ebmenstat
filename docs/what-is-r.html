1.1 What is R?

The most powerful and flexible statistical programming language in the world

— Norman Nie, Developer of SPSS.

R is in the first place a programming language, developed in 1993 by Ross Ihaka and Robert Gentleman, as an open source implementation of the S programming language developed by John Chambers and others, with further influences from the Scheme and Lisp programming languages. It is a so-called interpreted language, with its source code primarily written in C, Fortran and R itself. R is mainly used for statistical computing and graphics, but can also be used for importing/exporting data, creating interactive and dynamic reports and websites (“reproducible research”), and even for downloading and analyzing Twitter feeds!

As explained by John Chambers, the history of R illustrates its connection to data science. The S system aimed to provide an interactive environment for data analysis, providing convenience, completeness and extensibility, and being implementable with a relatively modest programming effort. This philosophy continues to drive the development of R, and explains the popularity of novel developments such as the tidyverse.

R is also a software environment that implements the R programming language. The standard version can be downloaded for free via CRAN, The Comprehensive R Archive Network: http://cran.r-project.org/. A popular alternative implementation is RStudio: http://rstudio.com/.

Finally, R is also a community. This community consists of the members of the R Core Team, responsible for developing and maintaining the basic R source code; the members of CRAN, responsible for managing the more than 20,000 R packages, collections of functions that significantly extend the basic R functionalities; and last but not least, the many thousands of useRs, who exchange information on R via online and offline discussion fora. Each year, an official gathering of R users is organized under the name useR!. More information about official R conferences is available via https://www.r-project.org/conferences/.

Information about the R User Group at Sciensano is available via http://intranet.sciensano.be/RUserGroup.

1.1.1 R vs other statistical software

Why use R and not any other statistical software? We are fully aware that R is not the only statistical software in the world, and that you can obtain equally valid results in other software such as SAS. In general R stands out for its free and open access nature, its increasingly rich extension and integration possibilities, and its highly active user community; major downsides include the steeper learning curve and performance issues when working with very large files.

Pros Cons
R
  • Free and open source
  • Increasing extension and integration options
  • Large user community
  • Commonly taught at universities
  • Steep learning curve
  • Performance issues when handling very large datasets
  • Stability/quality of little-used packages questionable
SAS
  • Stable and reliable
  • High adoption rate in major industries
  • Official support
  • Handling large datasets
  • Expensive, closed source
  • Extensions require commercial modules that not necessarily follow similar programming paradigms
  • Limited availability for advanced graphics
  • Requires more programming effort
Stata
  • Wide range of statistical functions
  • User interface and scripting options
  • Support and extension packages from the Stata community
  • Moderately expensive, closed source
  • Cumbersome integration with other software
  • Limited to a data set that is open simultaneously
SPSS
  • Easy to learn and use
  • Expandable via commercial modules
  • Extensive literature
  • Commonly taught at universities
  • Expensive, closed source
  • Limited functionality
  • Difficult to automate and integrate into other applications
